The Ultimate React Pattern Guide

Internal guide on industry-standard React folder structures and hooks.

1. Scalable Folder Structure

At UniBolt, we move away from organizing by type (putting all components in one folder) and move towards Feature-Based Architecture.

src/ ├── components/ # Shared UI (Buttons, Inputs) ├── hooks/ # Shared Hooks (useAuth, useTheme) ├── utils/ # Helpers (dateFormatter, api) ├── features/ # FEATURE MODULES │ ├── auth/ # Login/Register Logic │ │ ├── components/ │ │ ├── api/ │ │ └── AuthSlice.js │ ├── dashboard/ # Dashboard Logic │ │ └── DashboardPage.jsx │ └── profile/ └── App.js
💡 Why? Feature-based structure keeps related code together. If you delete the "Profile" feature, you just delete one folder, rather than hunting for files across 5 different directories.

2. The Custom Hook Pattern

Separate your UI (JSX) from your Logic (State & Effects). This makes components cleaner and logic reusable.

Before (Messy Component)

BAD
const UserProfile = () => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch("/api/user").then(res => ...);
  }, []);

  if (loading) return <Spinner />;
  return <div>{user.name}</div>;
};

After (Custom Hook)

GOOD
// 1. Extract Logic to Hook
const useUser = () => {
  /* ...state & effect logic here... */
  return { user, loading, error };
};

// 2. Component is pure UI
const UserProfile = () => {
  const { user, loading } = useUser();

  if (loading) return <Spinner />;
  return <div>{user.name}</div>;
};

3. Composition over Inheritance

Avoid building massive "God Components" with 50 props. Instead, use Children Prop to compose layouts.

PATTERN
const Card = ({ children, title }) => (
  <div className="card">
    <h2>{title}</h2>
    <div className="card-body">
      {children}
    </div>
  </div>
);

Android Publishing Guide

From APK generation to Google Play Store submission.

1. Signing Your App

Android requires all APKs to be digitally signed with a certificate before they can be installed.

TERMINAL
# Generate Keystore keytool -genkey -v -keystore my-upload-key.keystore \ -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
⚠️ Critical: Never lose your Keystore file. If you lose it, you cannot update your app on the Play Store.

2. Configuring Gradle

Edit your android/app/build.gradle file to include the signing configuration.

GRADLE
signingConfigs { release { storeFile file("my-upload-key.keystore") storePassword "******" keyAlias "my-key-alias" keyPassword "******" } } buildTypes { release { signingConfig signingConfigs.release } }

3. Generating the Bundle

We recommend uploading an Android App Bundle (.aab) rather than an APK, as it allows Google to optimize the download size for user devices.

TERMINAL
cd android ./gradlew bundleRelease

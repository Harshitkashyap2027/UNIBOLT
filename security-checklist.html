Web Security Checklist

Mandatory validation steps before deploying any Capstone Project.

⚠️ Deployment Rule: Projects failing items 1, 2, or 3 will be rejected automatically during code review.

1. Authentication & Session Management

  • Password Hashing: Never store plain-text passwords. Use bcrypt (cost factor 10+) or Argon2.
  • JWT Storage: Do not store Access Tokens in LocalStorage. Use HttpOnly cookies to prevent XSS theft.
  • Session Timeout: Ensure tokens expire (e.g., 15 mins for access, 7 days for refresh).

2. Input Validation (Injection Attacks)

  • SQL Injection: Use Parameterized Queries (Prepared Statements) or an ORM like Sequelize/Mongoose.
  • XSS (Cross-Site Scripting): Sanitize all user inputs. In React, avoid dangerouslySetInnerHTML.

3. Server Hardening

// Use Helmet for secure headers const helmet = require('helmet'); app.use(helmet()); // Rate Limiting const limit = rateLimit({ windowMs: 15*60*1000, max: 100 }); app.use(limit);
